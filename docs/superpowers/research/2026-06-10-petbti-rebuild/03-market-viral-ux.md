# 멍BTI 리빌드 — 한국 시장 × 바이럴 테스트 UX 리서치

> 2026-06-10, 멍BTI 리빌드 리서치 3/4. 한국 반려동물 성격 테스트 시장 + 바이럴 테스트 UX → 기능·콘텐츠·퍼널 전략. 출처는 본문 링크.

## 요약
1. 두 장르: **(A) 진단형**(보호자가 답함, 양육 솔루션 연결 — 포동 DBTI) vs **(B) 의인화 놀이형**(내가 답함, "내가 강아지라면" 공유 — 한화 멍BTI, ktestone). A는 전환에 강하고 B는 확산에 강함. 주오 목표(공유 도구 + 제품 퍼널)엔 **A를 본체로, B의 가벼운 UX·공유 장치를 이식**한 하이브리드.
2. 검증 스펙: **문항 10~14개(2분 내)**, 이미지 선택지+진행바, 결과는 **캐릭터+유형명+구체 행동 묘사+궁합(찰떡/상극)+유형별 추천템**. 중간 이탈은 0.3~0.5%, 병목은 시작 전(인트로).
3. 공유 실태: 카카오 버튼보다 **네이티브 공유·링크 복사가 10배**, 확산은 X(트위터) 압도, 인스타는 유입 전환 낮은 대신 **스토리 인증 이벤트**용. **유형별 OG 이미지**가 링크 미리보기 차별화 핵심.
4. 퍼널은 "결과 유형 ↔ 추천 혜택의 연결고리"가 자연스러움의 전부. W컨셉(유형→행운템→쿠폰, CTR 8.6%), 스낵팟 SPTI(유형→잘 맞는 스낵, 5일 860만), 방구석연구소×피피픽(반려견 테스트→인스타 인증→체험팩 추첨, 10만).

## 1. 한국 펫 테스트 사례

### 포동 DBTI (LG U+, 2022) — 시장 기준점
- 훈련 전문가 데이터 기반(1만~25만 출처 혼재). **4지표×2극=16유형**(야생성 C/W, 의존성 T/N, 관계성 내/외향, 활동성 모험/안주), CTIL 같은 4글자 코드.
- 문항 **60개(보도)** — 1분 바이럴이 아니라 **진단 포지션**. 결과에 행복도·보호자점수·활동에너지·교감능력·산책지수 수치 + 맞춤 양육법.
- 퍼널: 무료 DBTI → 성향 상담소(4개월 1,200건) → 훈련 클래스(90분 15만원) → 커뮤니티. **무료 진단 미끼 → 유료 교육**.
- 성과: 4개월 등록견 10만, 10개월 가입 20만. **왜 터졌나**: MBTI 문법 차용, "문제행동 교정" 아닌 "성향 이해" 따뜻한 프레임, 전문가 데이터 신뢰, 결과 지표가 보호자 대화 소재, 무료·저장벽.

### 기타
- **한화 LIFEPLUS 멍BTI(2020)**: 의인화 놀이형 10문항, "INTJ 페키니즈" MBTI×견종, PV 76만/UV 56.2만. → **'멍BTI'란 이름은 이미 "내가 강아지라면"으로 소비된 이력** → 주오는 "우리 강아지 진단"임을 인트로에서 즉시 드러내야.
- **방구석연구소×피피픽**: 10~13문항, 능력치+팁+궁합, **인스타 인증 시 추첨 간식 체험팩** → 10만 참여. 펫 커머스 연계 모범.
- **펫닥 펫 체크 마인드**: 대표 유형 + 항목별 정도 + 관리법, 라이트/본검사 이원화.
- **PurrJung(고양이 16유형)**: 12문항 + 고양이-인간 궁합, 가입 불필요.
- **봉봉(vonvon)**: 견주-강아지 매칭류 원조, 이미지+텍스트 카드 1~2분 SNS 포맷 시조.

## 2. 바이럴 테스트 UX 패턴

**인트로**: 제목 + 세계관 한 줄 카피 + **누적 참여자 수** + **예상 소요시간** + 반응 수 + 시작 버튼. 이탈 최고점은 **시작 페이지(3.29%)**, 문항 중반은 0.3~0.5% → 인트로 카피·소요시간 안내가 승부처.

**문항 sweet spot = 10~14개**: 멍BTI 10, W컨셉 11, SPTI/ktestone/PurrJung 12, 방구석 13, 미니맵 14, 포동 60(진단 예외). 16유형 만들려면 4축×3=12 최소. 진행바 + 한 화면 한 문항 공통. 문항 자체를 상황극으로 쓰면 문항이 콘텐츠가 됨.

**결과 대기 연출**: 효과 정량 1차 자료 없음(공백). 관행상 짧은 연출(2~3초)로 기대감 증폭, 5초 이상은 역효과. 주오는 광고 대신 **주멍이 브랜드 애니메이션** 권장.

**결과 페이지 요소**: 포토카드형 이미지(캐릭터+유형명+키워드), **궁합(찰떡/상극 쌍 → 친구 태그 트리거)**, **유형 희귀도 %**(주오는 D1 실집계로 "전체 참여견 중 N%" 가능 — 국내 차별화), 보조 수치 지표, 다시하기 + 다른 테스트 추천.

**공유 실태 (W컨셉×푸망 실측 — 핵심 데이터)**:
- 네이티브 공유 9,793 vs 카카오 버튼 859 — **10배**. 카카오 전용 버튼만 만들면 안 됨.
- 확산은 X 압도(노출 126.5만→플레이 1.8만), 인스타는 전환 약함(노출 3,262→414). 카카오 플러스친구 발송 전환 ~9.5%.
- **공유하는 유형 ≠ 전환하는 유형**(공유 1위 ESFJ는 광고 클릭 2.78%, 클릭 1위 ENTJ) → 유형별 CTA 강조 요소 차등 설계 근거.
- 참여 77% 여성, 18~24·45~54세까지 확산.

## 3. 공유 메커니즘 모범사례
- **유형별 OG 이미지**: 결과 페이지가 유형별 고유 URL + 그 유형 캐릭터·유형명 박힌 OG. 규격 **1200×630**(카카오 2:1, 500KB↓). 16유형 동적 생성(satori+sharp+CDN 캐싱)이 검증 패턴이나 8~16유형은 정적 제작도 현실적. 수정 후 [카카오 OG 캐시 초기화](https://developers.kakao.com/tool/clear/og).
- **카카오 피드 템플릿**: 이미지(2:1)+제목+설명+**버튼 2개**("결과 보기" + **"나도 테스트하기"**). 받은 사람 행동 경로를 버튼으로 분리.
- **인스타 스토리 9:16**: 인스타는 링크 전환 약하나 스토리 인증 문화 강함 → "스토리용 이미지 저장(1080×1920, @태그 자리)" 버튼 + 해시태그/캡션 한 번에 복사.
- **무엇이 유입을 끄나**: 신규 = 유형별 OG 링크 공유(X·카톡·커뮤니티), 재방문·팔로워 전환 = 스토리 인증 이벤트(브랜드 계정 태그→프로필→링크인바이오). 둘 다 필요.

## 4. 브랜드 퍼널
**자연스러운 방식 = "유형과 혜택 사이 연결고리"**:
- W컨셉: 유형→행운템→쿠폰, "광고 같지 않아서" CTR 8.6%(앱 평균 8배).
- 스낵팟 SPTI: 성격 결과 + "잘 맞는 스낵" — 식품 유형→제품 매핑 원형.
- 방구석×피피픽: 테스트→인스타 인증→**체험팩 추첨**(구매 아닌 체험으로 진입장벽↓).
- 포동: 무료 진단→유형별 양육 솔루션→상담→유료 클래스.

**거부감 주는 방식(반면교사)**: 결과 전 로그인/가입 강제, 유형 무관 일괄 광고, 과도한 데이터 수집 티내기.

## 5. 콘텐츠 톤
1. **구체 행동 묘사로 '어떻게 알았지' 인정 포인트**: "산책 중 다른 강아지 만나면 일단 멈춰서 3초 관찰부터".
2. **부정 특성은 긍정 프레임으로**: "소심하다" 쓰려면 무조건 "소심하지만 OOO".
3. **유형명에 캐릭터성·리듬**, 설명은 담백.
4. **공유 심리 = 정체성 확인받기**: 카드에 박히는 한 줄이 대화를 여는 말이 되게.
5. **주오 톤(브랜드 제약)**: 과한 감성팔이·의인화 배제, 행동 관찰 + 가벼운 유머 + 보호자에게 말 걸기. 예: "간식 봉지 소리에 0.5초 반응. 근데 막상 주면 일단 냄새 검사부터 하는 신중파. 손바닥에 올려 천천히 주면 더 좋아해요." (장면→인정→담백한 케어 팁, 펫 1인칭 화자 금지).

## 6. 확장 아이디어
- **견주-반려견 궁합**(봉봉·PurrJung 검증): 보호자 5문항+반려견 결과 결합 "우리 둘 케미 리포트". 기존 결과 데이터 재사용 → 효율 높음.
- **냥BTI**: 간식·베이커리 강아지·고양이 공용, 사료 종별 → 냥BTI는 키튼·캣 사료+공용 간식 퍼널 + 주냥이 마스코트 연결.
- **유형별 카드뉴스 16회분**: 테스트→피드→테스트 순환.
- **굿즈·오프라인**: 관광바우처 제휴처 현장 "멍BTI 부스 → 유형별 간식 샘플".
- **시즌 리스킨**: 명절 귀성(낯선 집), 여름 산책 등 결과 추천템·이벤트만 교체.

## 기능 체크리스트 (임팩트순)

### Must-have
1. 유형별 고유 결과 URL + 유형별 OG 이미지(1200×630)
2. 문항 10~14개, 한 화면 한 문항+진행바, 2분 내
3. 네이티브 공유 + 링크 복사 + 카카오 버튼 3종(플로팅 바)
4. 포토카드형 결과 이미지(캐릭터+유형명+한 줄 요약)
5. 궁합(찰떡/상극 유형)
6. 유형별 간식 추천 1~2종 + CTA(추천 이유 = 유형 해설 연장)
7. 다시하기 + 다른 테스트 출구
8. 로그인 없이 결과까지

### Nice-to-have
9. 인트로 누적 참여 수 + 소요시간
10. 유형 희귀도 %(D1 실집계)
11. 스토리용 9:16 이미지 + 해시태그/캡션 복사
12. 결과 대기 연출 2~3초(주멍이)
13. 보조 지표 2~3개(간식 취향·산책 텐션)
14. 다견용 "한 마리 더 검사하기"
15. 유형별 카드뉴스 재발행

> 16유형은 네이밍·일러스트·설명·추천템·OG가 16배 리소스. **8유형(3축)으로 시작해 반응 보고 16으로 쪼개는 단계적 접근도 합리적.**

## 결과 페이지 구성 권장안 (위→아래)
1. 포토카드(첫 화면 꽉 차게, 9:16/OG 원본) 2. 희귀도/참여 수 3. 유형 해설(장면 묘사 3 + 강점 + 긍정 변환 1, 담백) 4. 보조 지표 게이지 2~3 5. 궁합(찰떡/밀당) 6. **유형별 추천 간식(퍼널 본체, 추천 이유=해설 연장)** + 스토어 CTA 7. 공유 플로팅 바 8. 이벤트 박스(스토리 인증→체험팩) 9. 다시하기/다른 테스트(냥BTI·음식 검색기)

## 브랜드 퍼널 운영 원칙
1. 결과 상단은 끝까지 비상업 구역(신뢰=공유량). 2. **유형→제품 매핑표를 먼저 만들고 문항을 거꾸로 설계**(제품 구분에 필요한 행동 축을 문항으로 환원). 3. 유형별 공유율 vs 클릭률 분리 추적(D1+GA4 유형 코드별 이벤트).
