# 우리 아이 체크 도구 — 음식 안전 검색기 설계

- 작성일: 2026-06-09
- 대상 프로젝트: `linkinbio-web` (펫푸드주오 링크인바이오)
- 상태: 설계 확정(검토 대기) → 다음 단계 구현 계획(writing-plans)
- 연계 자산: 인스타 피드 "먹으면 안 되는 음식" 맛보기 카드뉴스(ig-feed로 별도 기획)

---

## 1. 배경 · 목표

인스타 피드에서 "강아지·고양이가 먹으면 안 되는 음식"은 저장·공유가 잘 터지고 신규 유입(top-of-funnel)이 강한 주제다. 피드 한 장에 다 담는 대신, **대표 음식 몇 개만 피드로 보여주고 → 링크인바이오의 검색기로 유입**시키는 퍼널을 만든다.

- 1차 목표: 유틸리티(진짜 도움)로 **프로필 방문·체류·재방문** 유도, "반려 라이프 친구" 포지션 강화
- 2차 목표: 유틸 → 브랜드 **소프트 루프백**(안심 간식 보기 등)으로 자연 연결
- 검색기는 한 번 만들면 계속 쓰는 **자산** (피드 글은 소모, 도구는 누적)

## 2. 범위 (v1)

**In scope**
- 음식 이름 검색 → **🟢먹어도 돼요 / 🟡조심해요 / 🔴안 돼요** 3단계 신호 + 한 줄 이유
- 대표 음식 **30~50개**로 시작, 이후 데이터만 추가하면 확장
- 강아지·고양이 **공용 한 묶음** (종 차이는 음식별 메모 한 줄)
- 링크인바이오 **바텀시트**로 구현 (기존 `StoreFinderSheet` 패턴 복제)
- `우리 아이 체크 도구` 섹션 신설 (추천 상품 아래)

**Out of scope (이번엔 안 함)**
- 적정 급여량·대체 간식 추천·제품 자동 매칭 (v2 후보)
- 강아지/고양이 종 토글 (대부분 음식 동일 → 공용으로 충분)
- 전용 URL 페이지·SEO 라우트 (`/petfoodjuo/food` 등 — v2에서 시트 컴포넌트 재사용해 승격 가능)
- 매장찾기 버튼을 도구 섹션으로 이동 (지금은 푸터 유지)

## 3. 사용자 흐름 (퍼널)

```
인스타 피드 "먹으면 안 되는 음식" (대표 5개, 주멍·주냥 카드뉴스)
   └ 캡션/마지막 컷 CTA: "이 외에 궁금한 음식? 프로필 링크 → '우리 아이 먹어도 돼요?'"
        └ 링크인바이오(/petfoodjuo) 진입
             └ '우리 아이 체크 도구' 섹션의 카드 탭
                  └ 음식 검색 바텀시트 (검색 → 판정)
                       └ (soft) 루프백: "안심하고 줄 수 있는 간식 보기" → 공식몰
```

## 4. 데이터 모델

음식 1개 = 아래 형태. `src/data/foodSafety.json` 에 배열로 저장하고, 가져올 때 `FoodSafetyItem[]`로 캐스팅한다(`storeLocations.json`을 `StoreLocation[]`로 쓰는 방식과 동일). JSON이라 사장님이 나중에 음식 추가하기도 쉽다. `FoodSafetyItem`/`FoodVerdict` 타입은 `FoodCheckSheet.tsx`(또는 별도 타입 파일)에 정의.

```ts
type FoodVerdict = "safe" | "caution" | "danger";

type FoodSafetyItem = {
  id: string;            // "chocolate"
  name: string;          // "초콜릿"  (대표 표기)
  aliases: string[];     // ["초콜렛","쵸콜릿","카카오","코코아","choco"]  검색 매칭용
  emoji?: string;        // "🍫"  (선택, 빠른 시각 신호)
  verdict: FoodVerdict;  // safe | caution | danger
  reason: string;        // 한 줄 이유 (효능·치료 표현 금지, 사실 위주)
  note?: string;         // 종 차이/조건 한 줄 ("다크초콜릿일수록 더 위험해요")
};
```

- **판정 3단계**
  - 🔴 `danger` = **안 돼요** — 중독·위험, 절대 금지
  - 🟡 `caution` = **조심해요** — 소량·조건부, 권장 안 함
  - 🟢 `safe` = **먹어도 돼요** — 간식으로 소량 OK
- **공용**: 종 차이는 `note`에만 ("고양이는 특히…")
- `aliases`가 검색 핵심 — 오타·영문·줄임말 대응

## 5. 음식 데이터 (검수 전 초안 — 발행 전 검수 필수)

아래는 시작용 대표 목록. **신뢰 출처(ASPCA, AAHA/VCA 등 수의 자료)** 기반으로 구현 시 한국어로 정리하고, **발행 전 사장님(또는 협력 수의사) 1회 검수**한다. 추측으로 채우지 않는다.

- 🔴 안 돼요: 초콜릿, 양파, 마늘, 파, 부추, 포도, 건포도, 자일리톨(껌·사탕), 카페인(커피), 알코올, 마카다미아, 복숭아씨·자두씨(씨앗), 생빵반죽(이스트)
- 🟡 조심해요: 우유·유제품, 치즈, 생계란, 짠 음식·가공육(햄·소시지), 익힌 뼈, 견과류, 감귤류, 덜 익은 토마토·줄기, 아보카도, 새우·오징어 다량
- 🟢 먹어도 돼요: 당근, 사과(씨 제거), 고구마(익힌), 단호박, 블루베리, 수박(씨·껍질 제거), 바나나, 오이, 무염 닭가슴살, 플레인 요거트(소량), 익힌 브로콜리(소량), 딸기

> ⚠️ 일부 항목은 종·양·조리 상태에 따라 판정이 갈린다(예: 유제품=고양이 유당불내, 아보카도=논쟁). 보수적으로 잡고 `note`로 보완. 최종 데이터는 구현 태스크에서 출처 명기 + 검수.

## 6. 검색 · 화면 동작

- **상단**: 검색창 + 필터칩 `[전체 / 🔴안돼요 / 🟡조심 / 🟢괜찮아요]` (매장찾기 칩과 동일 스타일)
- **검색 매칭**: `name` + `aliases` 부분일치, `useDeferredValue`로 입력 디바운스 (StoreFinder와 동일)
- **기본(검색 전)**: 전체 목록 노출, **위험한 것부터(danger→caution→safe)** 정렬 → 그냥 훑어도 되는 "작은 사전"
- **결과 카드**: 색상 신호(좌측 컬러 바/배지) + emoji + 음식명 + 한 줄 이유 + (note). 네오브루탈 카드(border-2, 하드 섀도)
- **결과 없음**: "아직 등록 안 된 음식이에요. 헷갈리면 수의사와 상담하세요."

색상 매핑(브랜드 톤 유지):
- danger 🔴 coral 계열 강조(브랜드 `#ff6b6b`) / caution 🟡 amber / safe 🟢 emerald

## 7. 안전 · 법적 카피 (그대로 사용)

브랜드 redline(효능·치료·예방·공포 마케팅 금지)과 sensitive-claims 원칙 준수.

- **시트 상단 상시 안내(작게)**: "일반적인 정보예요. 진단·처방이 아니며, 먹은 게 의심되거나 증상이 있으면 바로 수의사와 상담하세요."
- **🔴 위험 카드 추가 한 줄**: "이미 먹었다면 즉시 동물병원에 연락하세요."
- **시트 하단 면책**: "참고용 정보이며, 우리 아이 상태·기저질환에 따라 다를 수 있어요."
- 톤: 사실 위주, 공포 X. 브랜드 푸터 문구 "사람이 먹을 수 없는 건, 주지 않습니다"와 결 일치.

## 8. 화면 배치 (placement)

신규 섹션 **`우리 아이 체크 도구`**, 위치는 **추천 상품 바로 아래**.

페이지 순서:
1. 히어로
2. 추천 상품
3. **우리 아이 체크 도구 (신규)** ← 음식 검색기 진입 카드
4. 진행중인 이벤트 (멍BTI)
5. 주오 멤버십
6. Juo Company
7. 푸터 (매장찾기 + 문구)

- 멍BTI(기간 한정 **이벤트**)와 음식 검색기(상시 **도구**)는 성격이 달라 섹션 분리
- 진입 카드는 멍BTI와 같은 **featureCard** 스타일(배지 + 아이콘/일러스트 + 화살표). 아이콘 자리에 **주멍/주냥 일러스트** 활용 가능(캐릭터 결 연결)
- 도구가 늘면 이 섹션에 누적(향후 매장찾기도 이전 가능)

## 9. 아키텍처 · 파일 변경

기존 `StoreFinderSheet` → `LinkInBioPage`(상태/트리거) → `petfoodJuo.ts`(설정) 패턴을 그대로 따른다.

**신규**
- `src/data/foodSafety.json` — 음식 데이터(검수 후 채움)
- `src/components/FoodCheckSheet.tsx` — 검색 바텀시트. `StoreFinderSheet` 복제 후 지도/카카오 관련 제거, 결과를 판정 카드로 교체. `trackEvent` 동일 사용. `onClose` prop.

**수정**
- `src/data/linkPages/types.ts`
  - `FeatureCard`에 선택 필드 `opens?: "foodCheck"` 추가 (시트 트리거용 — 기존 `LinkAction.toggleCenterMap` 선례와 동일 컨셉)
- `src/data/linkPages/petfoodJuo.ts`
  - `sections`에 `featureCards` 타입의 `우리 아이 체크 도구` 섹션 추가(추천 상품 다음). 카드 1개에 `opens: "foodCheck"`, 배지(예: `무료 체크`), 아이콘/이미지
- `src/features/linkinbio/LinkInBioPage.tsx`
  - `const [showFoodCheck, setShowFoodCheck] = useState(false)`
  - `FeatureCards`/`FeatureCardButton` 클릭 시 `card.opens === "foodCheck"`면 `setShowFoodCheck(true)` (핸들러를 섹션 렌더러로 전달)
  - `<AnimatePresence>`에 `{showFoodCheck ? <FoodCheckSheet onClose={() => setShowFoodCheck(false)} /> : null}` 추가

> 트리거 배선은 `toggleCenterMap`(LinkAction)이 `onCenterMapToggle`로 시트를 여는 방식과 동일 패턴. `FeatureCard`에도 같은 식으로 얇게 추가.

## 11. 연계: 피드 맛보기 글 (별도 기획)

- 검색기 스펙 확정 후 **ig-feed**로 "먹으면 안 되는 음식" 카드뉴스 기획안 작성(파일 분리)
- 구성: 대표 위험 음식 ~5개, 주멍·주냥 리액션, 공용 Standard Closing Cover
- CTA: "이 외에 궁금한 음식? 프로필 링크 → '우리 아이 먹어도 돼요?' 검색기"
- 검색기 ↔ 피드는 한 세트로 운영(같은 카피 키워드)

## 12. 향후(v2 후보)

- `lovejuo` 등 다른 브랜드 페이지에서도 재사용
